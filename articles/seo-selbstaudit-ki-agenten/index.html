Why We Audit Ourselves

Das Prinzip dahinter ist unser Motto: Mensch mit Maschine schlägt sowohl Mensch als auch Maschine. Die Agenten crawlen, zählen und messen – schneller und gründlicher, als es ein Mensch an einem Nachmittag könnte. Der Mensch priorisiert, entscheidet und gibt frei. Auch diesen Artikel.

The principle behind it is our motto: human with machine beats both human and machine. The agents crawl, count and measure – faster and more thoroughly than a human could in an afternoon. The human prioritizes, decides and approves. Including this article.

And because an audit that only reports wins is not an audit: we are publishing the weaknesses too. 7.9/10 is not 10/10. That is exactly the point.

Prüffeld 1: Content-SEO mit SMarTrContent

Audit Domain 1: Content SEO with SMarTrContent

Unser Content-Agent SMarTrContent hat alle 20 Blog-Artikel durchleuchtet. Gesamtscore: 7,9/10. Die Einzelbefunde:

Was sitzt

  • 20/20 Artikel mit sauberer H1-Struktur. Genau eine Hauptüberschrift pro Artikel und Sprache.
  • 20/20 Artikel mit gültigem JSON-LD. Schema.org-Markup auf jeder Seite, maschinenlesbar für Suchmaschinen.
  • 200/200 Bilder mit Alt-Text. Kein einziges Bild ohne Beschreibung.
  • Sauberes Keyword-Mapping. Ein Haupt-Keyword pro Artikel, keine Kannibalisierung zwischen den Seiten.

Was nicht sitzt

  • Alt-Texte zu generisch. Vorhanden ist nicht dasselbe wie gut. „KI-Agent Bild" hilft niemandem – konkrete Beschreibungen fehlen vielerorts.
  • 1 Meta-Description fehlte komplett. Peinlich, aber gefunden.
  • 5 Meta-Descriptions zu lang (über 160 Zeichen) – Google schneidet sie in den Suchergebnissen ab.
  • 5 deutsche Artikel ohne interne Links. Sackgassen im eigenen Blog. Dieser Artikel macht es übrigens gerade anders.

Our content agent SMarTrContent examined all 20 blog articles. Overall score: 7.9/10. The individual findings:

What holds up

  • 20/20 articles with clean H1 structure. Exactly one main heading per article and language.
  • 20/20 articles with valid JSON-LD. Schema.org markup on every page, machine-readable for search engines.
  • 200/200 images with alt text. Not a single image without a description.
  • Clean keyword mapping. One main keyword per article, no cannibalization between pages.

What does not hold up

  • Alt texts too generic. Present is not the same as good. "AI agent image" helps nobody – specific descriptions are missing in many places.

Die ehrliche Liste: Was als Nächstes passiert

The Honest List: What Happens Next

Ein Audit ohne Konsequenzen ist Dekoration. Das hier ist unsere Arbeitsliste – in dieser Reihenfolge:

  1. Prerendering gegen den leeren SPA-Body. Der Hauptgrund, warum Brave und DuckDuckGo unsere Seite kaum zeigen: Crawler ohne JavaScript sehen fast nichts. Statische HTML-Snapshots lösen das. Höchste Priorität.
  2. Meta-Descriptions fixen. Die fehlende ergänzen, die fünf zu langen kürzen.
  3. Interne Links ergänzen. Die fünf verwaisten DE-Artikel bekommen Anschluss.
  4. Sitemap-Trailing-Slashes vereinheitlichen. Klingt pedantisch, verhindert aber Duplicate-Content-Signale.
  5. hreflang einführen. Damit Suchmaschinen Deutsch und Englisch sauber auseinanderhalten.
  6. Google Search Console und Bing einrichten. Schluss mit Blindflug.

Jeder Punkt wird von den Agenten vorbereitet und vom Menschen freigegeben. Wie alles hier.

An audit without consequences is decoration. This is our work list – in this order:

  1. Prerendering to fix the empty SPA body. The main reason Brave and DuckDuckGo barely show our site: crawlers without JavaScript see almost nothing. Static HTML snapshots solve this. Top priority.
  2. Fix meta descriptions. Add the missing one, shorten the five that are too long.
  3. Add internal links. The five orphaned German articles get connected.
  4. Unify sitemap trailing slashes. Sounds pedantic, but prevents duplicate content signals.
  5. Introduce hreflang. So search engines can cleanly tell German and English apart.
  6. Set up Google Search Console and Bing. No more flying blind.

Every item is prepared by the agents and approved by a human. Like everything here.

Das SPA-Problem im Detail

Die Hauptseite rendert client-seitig: Der initiale HTML-Response enthält im Wesentlichen ein leeres Root-Element plus Script-Tag. Googlebot rendert JavaScript (verzögert), aber viele Crawler – darunter die von Brave Search und DuckDuckGo genutzten Quellen – indexieren primär den initialen HTML-Body. Ergebnis: Die Seite existiert für sie kaum. Fix: Prerendering bzw. statische Snapshots pro Route, ausgeliefert an Crawler oder als Build-Artefakt für alle. Die statischen Blog-Seiten wie diese hier sind davon nicht betroffen – sie liefern vollständiges HTML im ersten Response.

The SPA problem in detail

The main site renders client-side: the initial HTML response essentially contains an empty root element plus a script tag. Googlebot renders JavaScript (with delay), but many crawlers – including the sources used by Brave Search and DuckDuckGo – primarily index the initial HTML body. Result: for them, the site barely exists. Fix: prerendering or static snapshots per route, served to crawlers or built as static artifacts for everyone. Static blog pages like this one are not affected – they deliver complete HTML in the first response.
